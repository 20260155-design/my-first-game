# 4주차 실습 정리

## 오늘 한 것
- 원형 / AABB / OBB Bounding Box 시각화
- 세 방식의 충돌 판정 차이 관찰

## 관찰 내용

### 원형 VS AABB
- 원형은 오브젝트를 원으로 감싸는 방식으로 회전할 때 박스의 형태나 크기에 변화없음. -> 회전 연산을 완전히 무시 가능할 있어 동적인 물체에 매우 효율적이지만, 긴 오브젝트의 경우 포함되지 못한 부뷴이 남게된다.
- 원형의 충돌 방식: 두 원의 중심 사이의 거리가 주 원의 반지름의 합보다 작으면 충돌한다.(루트는 연산 비용이 크기 때문에 제곱으로 비교하는 것이 효율적이다.)
- AABB는 'AABB VS OBB' 참조.
- 모든 축(x, y)에서 두 사각형의 범위가 동시에 겹쳐저야 충돌한다. (단순 크기 비교 연산만을 사용하므로 연산 비용이 적게 든다.)

### AABB VS OBB
- AABB는 오브젝트가 회전하더라도 오브젝트를 감싼 박스가 회전하지 않음, 대신 박스의 크기가 재조정됨. -> 계산이 매우 단순하고 성능이 뛰어나지만, 오브젝트가 대각선으로 기울어지면 불필요한 공백이 박스에 포함되어 정밀도가 하락함.
- OBB는 오브젝트가 회전하면 오브젝트를 감싼 박스도 같이 회전함. -> 오브젝트를 AABB에 비해 횔씬 밀착하여 감싸기 때문에 정밀도가 높지만, 계산이 복잡하고 수학 연산이 추가로 필요함.

## 내 게임에 적용한다면
- AABB가 적합할 것 같다. AABB는 오브젝트를 사각형의 형태로 감싸는 구조의 박스를 의미 한다. 내 게임은 픽셀 도트형식의 2D게임으로 평면내에서만 움직이므로 AABB와 잘어울린다. 또한, AABB는 그리드나 타일 맵 구조에 적합하고 연산 효율이 높기 때문에 수 많은 오브젝트를 처리 가능하다. 따라서 높은 'AABB는 2D 게임의 특성을 살리면서, 계산 비용을 최소화하여 성능을 극대화할 수 있는 가장 잘맞는 충돌 처리 방식'이다.
