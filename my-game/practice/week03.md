# Week 3 실습 기록

## 목표
플레이어 캐릭터를 키보드로 움직이기 구현

## AI 대화 기록

**Q1: Pygame에서 창을 지정하고 싶어.**

- AI 답변: Pygame에서 창 크기를 지정하는 (width, height)는 수학의 좌표계와 같다고 생각하면 쉽습니다. 하지만 우리가 흔히 아는 수학 좌표와는 방향이 조금 다릅니다.
- width (가로): 창의 왼쪽 끝에서 오른쪽 끝까지의 길이를 픽셀(Pixel) 단위로 정합니다.
- height (세로): 창의 위쪽 끝에서 아래쪽 끝까지의 길이를 픽셀 단위로 정합니다.
- Pygame에서 위치를 계산할 때 꼭 알아야 할 특징이 있습니다.
기준점 (0, 0): 창의 왼쪽 상단 구석이 시작점입니다.
- X축 (가로): 오른쪽으로 갈수록 숫자가 커집니다.
- Y축 (세로): 아래쪽으로 갈수록 숫자가 커집니다. (수학 시간에는 위로 갈수록 커지지만, 컴퓨터 그래픽에서는 아래로 갈수록 커지는 경우가 많습니다.)
- 적용 결과: width, height를 이용해 가로의 길이를 800, 세로의 길이를 600으로 지정함.

**Q2: 창이 사라지지 않고 계속 실행되게 해줘.**

- AI 답변: 창이 뜨자마자 사라지지 않고 계속 정상적으로 실행되게 하려면, 프로그램이 종료되지 않도록 붙잡아두는 **'무한 루프(Infinite Loop)'**와 사용자의 명령을 듣는 **'이벤트 처리'**가 반드시 필요합니다. 이를 보통 **게임 루프(Game Loop)**라고 부릅니다.
- while running(무한 루프): 컴퓨터에게 "내가 그만하라고 할 때까지 이 안의 코드를 계속 반복해!"라고 시키는 것입니다. 이 루프 덕분에 창이 계속 화면에 떠 있게 됩니다.
- pygame.event.get()(이벤트 수집): 윈도우 운영체제는 프로그램에게 계속 질문을 던집니다. "너 지금 살아있니?", "사용자가 클릭했는데 알고 있니?" 이 함수를 호출해서 쌓여있는 질문(이벤트)들을 처리해줘야만 윈도우가 '응답 없음' 표시를 띄우지 않고 정상 실행 중이라고 판단합니다.
- pygame.QUIT(종료 신호): 창의 오른쪽 상단 [X] 버튼을 누르는 동작입니다. 이 조건을 체크해서 running을 False로 바꿔줘야만 프로그램이 깔끔하게 종료됩니다.
- 적용 결과: 무한루프를 구축하여 정상적이고 지속적으로 프로그램을 실행할 수 있게됨, 프로그램을 종료할 때, (응답없음)이 뜨지 않음.

**Q3: 캐릭터를 설정하고(만들고) 실현하고 싶어.**
- 색상 (Color): RGB 방식을 사용합니다. (Red, Green, Blue) 순서로 0~255 사이의 숫자를 조합합니다.
- 위치 (Position): 창의 왼쪽 상단을 (0, 0)으로 잡고, 가로(x)와 세로(y) 좌표를 정합니다.
- 크기 (Size): circle_radius 변수 값을 조절하여 반지름의 크기를 정합니다.
- 이동(speed): base_speed 값을 통해 속도를 정할 수 있습니다.
- pygame.draw.circle() 함수를 사용하여 나타낼 수 있습니다. 'pygame.draw.circle(화면, 색상, (중심x, 중심y), 반지름)'
- 적용 결과: 회색(200, 200, 200)의 색상의 반지름 값 25인 구 형태의 캐릭터를 만들었고, base_speed의 값을 5로 정하였다.
